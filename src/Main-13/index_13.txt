1-13. 선행처리기

 # define ?
 
1. 선행처리기
 (1) 개념
  - 컴파일에 앞서 프로그램 상단에 선언된 지시자들 미리 처리하는 역할
  - 소스 프로그램을 컴파일러에 전달전에 사용자가 정의한 각종내용을 먼저 처리함
  
 (2) 특징
  - #으로 시작함
  - 한줄에 하나의 명령만 씀
  - 명령 끝에 세미콜론(;) 안붙임
  - 일반적으로 첫칸부터 시작
 
 (3) 종류
  - #include : 파일을 포함하는 기능
  - #define : 매크로 정의
  - #if, #else, #elif, #endif : 조건부 컴파일 (시험에안나와)
  
2. 매크로
 (1) 개념
  - 선행처리기에서 #define을 사용해서 단순 치환되는 자료
  - 프로그램 작성시 명령,수식 또는 상수값이 자주 사용될때 정의해서 사용
  
 (2) 종류
  - 매크로 상수
    * 미리 정의한 매크로 상수명이 프로그램에서 사용되면, 
	  매크로 확장 문자열로 치환
	  ex) define N 10  (N에는 10이라는 값이 대입되고
						앞으로사용되는 모든 N이라는걸 사용할때
						10으로 치환해주는것.)
  - 매크로 함수
    * 전달의 인자의 형태를 가지고 치환하는 동작을 말함
	  ex) SQR(x)x*x (함수명이 있고, 인자값이 있고 그안에서 뭔가 동작되는것을 정의하는것)
  
3. 조건부 컴파일
 (1) 개념
   - if
   
 (2) 조건부 컴파일 구문
   - #if 조건 : 조건 참일때
   - #else : 조건이 거짓일때 컴파일 수행될 문장
   - #elif : else와 if가 같이 있는 문장(else if역할)
   - #endif : 조건의 끝
