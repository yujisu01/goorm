1-20. 프로세스 생성 
- 프로그램 설치하면 하드디스크 저장- 그리고 cpu가 처리하는 도구를 프로세서 라고 함
- cpu와 하드디스크상 데이터를 주고받지 못함 (속도차이 많이나서)
  -> 그래서 주기억장치라는것을 중간에 둠(램)
  -> 프로그램 클릭시, 필요로 하는 파일들을 주기억장치에 적재함 
  -> 실행중인 프로그램은 프로세스 
  -> 프로세스는 각각의 프로세스의 id인 PID를 부여받게 된다
  // 프로그램 클릭->주기억장치->프로세스가 됨->이 프로세스를 프로세서가 처리하면서 사용할수 있게됨
  
  1. fork
   (1) fork 개념
    - 프로세스를 생성하고자 할때 사용함
	- fork호출이 성공하면 부모프로세스 에서는 자식프로세서의 PID를 반환받고,
	  실패시에는 -1을 반환한다.
	- 자식 프로세스는 0값을 리턴받는다.
	- unistd.h 파일을 include 해야한다. (uniInstalled.h파일)
	
	// 이것만 일단 알면됨
	- fork함수를 쓰게되면 3가지의 리턴값이 나옴 (하나는 음수, 하나는 0, 하나는 양수)
	  * 음수: 실패시
	  * 0: 자식프로세스
	  * 양수: 부모프로세스
